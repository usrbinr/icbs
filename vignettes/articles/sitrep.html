Debugging Layouts with story_sitrep()

Introduction

When building complex story layouts with story_layout(), it can be difficult to understand how the various components fit together. The story_sitrep() function provides a “situation report” that shows you exactly what’s in your layout before you render it.

This is useful for:

  • Debugging - Understanding why elements overlap or look wrong
  • Planning - Previewing layout proportions before committing
  • Documentation - Seeing all parameters at a glance

Basic Usage

story_sitrep() takes the same arguments as story_layout() and returns a diagnostic object:

sitrep <- story_sitrep(
    title = "My Report Title",
    subtitle = "Supporting context here",
    narrative = "Key insights and analysis",
    caption = "Source: Data Team"
)

The output shows:

  1. Components - Which elements are included, their type (text or ggplot), and size in points
  2. Layout Proportions - How much vertical space each element takes
  3. Layout Diagram - ASCII visualization of the structure
  4. Tips - SWD-inspired suggestions for improvement

Understanding the Layout Diagram

The ASCII diagram shows how your layout will be structured. When you have both a plot and narrative, the diagram shows their arrangement:

# Create a placeholder plot for demonstration
p <- ggplot(mtcars, aes(x = factor(cyl), y = mpg)) +
    geom_col() +
    theme_minimal()

# Layout with narrative on the RIGHT

    plot = p,
    title = "Sales Analysis",
    narrative = "Commentary here",
    narrative_position = "right"

Workflow: Sitrep Then Build

A recommended workflow is to check your sitrep before rendering:

# Step 1: Check the sitrep
my_sitrep <- story_sitrep(
    title = "**Q4 Results: {#E63946 Sales} Exceed Target**",
    subtitle = "Regional performance summary",
    narrative = "Key findings go here",
    narrative_position = "right",
    narrative_width = 0.35,
    caption = "Source: Finance"
)

requiring attention in Q1.",
    narrative_position = "right",
    narrative_width = 0.35,
    caption = "Source: Finance"
)

Accessing Sitrep Data Programmatically

The sitrep is an S7 object. You can access its properties directly:

sitrep <- story_sitrep(
    title = "Report",
    subtitle = "Details",
    narrative = "Analysis"
)

# Access the components table
sitrep@components
  component present type size_pt
1     title    TRUE text      16
2  subtitle    TRUE text      11
3      plot   FALSE <NA>      NA
4 narrative    TRUE text      10
5   caption   FALSE <NA>      NA
# Access layout proportions
sitrep@layout$content_height
[1] 0.78
# Check for tips
sitrep@tips
[1] "Use **bold** in narrative to emphasize key takeaways and action items"       
[2] "Use auto_heights=TRUE in story_layout() to automatically size title/subtitle"

Preventing Text Clipping with auto_heights

A common issue is titles or subtitles getting clipped when they wrap to multiple lines. The auto_heights parameter (default TRUE) automatically calculates appropriate heights:

# Long title that wraps - auto_heights handles it automatically
story_layout(
    plot = p,
    title = "**{#E63946 10 warehouses} need immediate attention for error rates**",
    subtitle = "What are the next steps at our highest volume sites?",
    caption = "Source: Operations Dashboard",
    auto_heights = TRUE,  # Default - calculates heights from text length
    output_width = 12     # Match your ggsave width for accurate estimates
)

For manual control, set auto_heights = FALSE and specify heights explicitly:

story_layout(
    plot = p,
    title = "Short title",
    auto_heights = FALSE,
    title_height = 0.10,
    subtitle_height = 0.06
)

Best practice: Keep titles under 60 characters. Concise, punchy titles are both easier to read AND less likely to have layout issues.
